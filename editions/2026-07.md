## Key findings from June's open track

### Finding 1: Timestamp visibility is worth ~37 temporal points

plainfile and obsidian use the same naive keyword search. The only difference: obsidian inlines timestamps into the retrieved text, so the answering model can see dates. Result: obsidian scores **0.375** on temporal questions, plainfile scores **0.000**.

**Implication:** If you build or operate agent memory, putting timestamps inside the retrieved text is the cheapest temporal-reasoning upgrade available — no graph, no extraction, no LLM in the write path.

### Finding 2: Basic Memory clears the naive-RAG baseline

**0.335 answerable** edges past the naive-RAG bar (0.300) — the first tool to justify its machinery. The margin is narrow (3.5 points), so the main-track Claude results will be decisive.

### Finding 3: The open track is consistent with the main track

The no-memory canary scored 0.000 answerable in both tracks. The benchmark does not leak, and the open-model track produces a clean canary.
